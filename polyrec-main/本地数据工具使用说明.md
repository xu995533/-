# Polymarket BTC 本地数据工具使用说明

这个工具的作用很简单：把 Polymarket 的 BTC Up or Down 历史价格曲线下载到本地电脑里，后面做回测、统计、画图时不用每次重新联网抓数据。

目前支持三类市场：

- `1h`：BTC Up or Down 每小时
- `4h`：BTC Up or Down 4小时
- `daily`：BTC Up or Down 日线

## 一、最简单用法

进入这个文件夹：

```text
polyrec-main
```

直接双击：

```text
data_cache.py
```

会出现菜单：

```text
1. 查看本地数据数量
2. 下载/更新最新数据
3. 导出本地数据到 CSV
4. 退出
```

输入数字，然后按回车即可。

## 二、三个快捷按钮

文件夹里还有三个 `.bat` 文件，可以直接双击：

```text
data_cache_status.bat
```

查看本地已经有多少数据。

```text
data_cache_sync_240.bat
```

下载/更新数据。默认检查最近 240 个小时线市场、最近 240 个 4小时市场，以及能找到的日线市场。

```text
data_cache_export.bat
```

把数据库里的数据导出成 CSV 文件。

## 三、数据存在哪里

最重要的数据在这里：

```text
local_data\polymarket_btc.sqlite
```

这是 SQLite 数据库，是核心数据文件。

导出的 CSV 在这里：

```text
local_data\exports\
```

里面通常会有：

```text
btc_1h_price_points.csv
btc_4h_price_points.csv
btc_daily_price_points.csv
```

## 四、进度怎么看

下载时会看到类似内容：

```text
[1h] 1/240 跳过，已有完整数据
[4h] 2/240 已保存：btc-updown-4h-...
[daily] 29/240 没找到这个市场
```

含义如下：

- `已保存`：这条市场数据已经下载并写入本地数据库。
- `跳过`：本地已经有完整数据，不重复下载。
- `没找到这个市场`：Polymarket 没有这个历史市场，常见于更早的 daily 日线。
- `完整曲线`：UP 和 DOWN 两边都有价格曲线。

如果看到很多 `跳过`，不是坏事，说明本地已经有这些数据。

## 五、当前已验证的数据量

截至这份说明创建时，本地数据库已经有：

```text
1h    240 条完整市场
4h    240 条完整市场
daily 100 条完整市场
```

Daily 没有 240 条，是因为 Polymarket 更早的日线市场接口查不到，不是程序没跑完。

## 六、命令行用法

如果会用命令行，也可以在 `polyrec-main` 文件夹里运行：

```powershell
python data_cache.py status
```

查看本地数据状态。

```powershell
python data_cache.py sync --limit 240
```

下载/更新最近数据。

```powershell
python data_cache.py export
```

导出 CSV。

只更新某一种周期：

```powershell
python data_cache.py sync --timeframes 1h --limit 240
python data_cache.py sync --timeframes 4h --limit 240
python data_cache.py sync --timeframes daily --limit 240
```

## 七、换电脑或拷到 U 盘

可以把整个 `polyrec-main` 文件夹复制到 U 盘或另一台电脑。

重点是不要只复制 `data_cache.py`，要把整个文件夹一起复制，因为数据库在：

```text
polyrec-main\local_data\polymarket_btc.sqlite
```

到新电脑后，双击：

```text
data_cache_status.bat
```

如果能看到 `1h / 4h / daily` 的数据数量，说明数据库正常。

## 八、注意事项

1. 下载数据需要联网。
2. 查看状态和导出 CSV 不一定需要联网，只要本地数据库还在。
3. 正在下载时不要直接删除或移动 `local_data` 文件夹。
4. 如果窗口停在“按任意键继续”或“按回车退出”，说明当前任务已经结束。
5. 后续做回测时，优先读取 `polymarket_btc.sqlite`，CSV 主要用于人工查看或临时分析。
